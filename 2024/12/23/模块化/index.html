模块化

:::info
Node.js 有两个模块系统：CommonJS 模块和 ECMAScript 模块。

官方文档解释:CommonJS 模块 | Node.js v22 文档

:::

1.模块化概述

1.1.什么是模块化?

·将程序文件依据一定规则拆分成多个文件，这种编码方式就是模块化的编码方式。

·拆分出来每个文件就是一个模块，模块中的数据都是私有的，模块之间互相隔离。

·同时也能通过一些手段，可以把模块内的指定数据“交出去”，供其他模块使用。

1.2.为什么需要模块化?

随着应用的复杂度越来越高，其代码量和文件数量都会急剧增加，会逐渐引发以下问题:

1.全局污染问题(命名冲突)

2.依赖混乱问题

3.数据安全问题

2.有哪些模块化规范

随着时间的批移，针对 JavaScript 的不同运行环境，相出现了多种模块化规范，按时间排序，分别为:

:::info
1.CommonJs– 服务端应用广泛

2.AMD

3.CMD

4.ES6 模块化 – 浏览器端应用广泛

:::

3.导入和导出的概念

模块化的核心思想就是:模块之间是隔离的，通过导入和导出进行数据和功能的共享

:::info
导出:模块公开其内部的一部分(如变量、函数等)，使这些内容可以被其他模块使用。

导入:模块引用和使用其他模块导出的内容，以重用代码和功能。

:::

导出数据

在 CommonJs 标准中，导出数据有两种方式

:::info
第一种方式:module.exports =value

第二种方式:exports.name =value

:::

2.无论如何修改导出对象，最终导出的都是 module.exports的值

5.ES6 模块化规范

ES6 模块化规范是一个官方标准的规范，它是在语言标准的层面上实现了模块化功能，是目前最流行的模

块化规范，且浏览器与服务端均支持该规范。

1.ESM 自动采用严格模式，忽路use strict’

2.每个 ES Module 都是运行在单独的私有作用域中

3.ESM 是通过 CORS 的方式请求外部JS 模块的

4.ESM 的 script 标签会延迟执行脚本

ESM 对外暴露的是引用关系,且只读
