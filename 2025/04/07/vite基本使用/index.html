vite基本使用

认识vite

Vite 是一个现代化的前端构建工具

  • 定义：Vite 旨在利用现代浏览器的原生 ES 模块支持，提供快速的开发体验，由开发服务器和构建命令两部分组成。开发服务器基于原生 ES 模块，提供超快的热更新(ESBuild)；构建命令使用 Rollup 打包代码，生成适用于生产环境的优化静态资源。
  • 优势：与传统构建工具如 Webpack 相比，Vite 具有极速启动、快速热更新、丰富的功能以及高度可扩展等优势。它解决了传统构建工具启动速度慢、热更新慢和配置复杂的痛点。

在Webpack中,运行本地服务本质上是打开一个本地服务器,用本地服务器去访问内存中打包转换后的代码(在内存中进行构建处理后的代码,并未真正打包),然后运行在浏览器上,当开发代码发生变化时,对应的打包后的文件也发生变化,本地服务器在去拿到更新后的代码,通过 HMR 或自动刷新机制更新浏览器内容

而Vite则不会对项目中所有的代码进行构建,Vite会开启一个本地开发服务器,此服务器具备静态文件服务和处理模块请求的能力(使用ESBuild进行代码转化 快10~100倍),开发阶段，服务器会直接将项目中文件以 ES 模块的形式提供给浏览器,浏览器在解析时遇到 ES 模块导入会向服务器按需对相应模块进行编译和转换，然后返回给浏览器,这种方式避免了对整个项目进行提前打包，显著加快了启动速度

Vite 运行本地服务的核心优势在于利用现代浏览器的 ES 模块支持，实现了按需加载和快速热更新,提升了启动速度

vite的基本使用

如果在代码中已经提供模板和逻辑代码,直接进行 npx vite 运行项目即可

  • **vite可以直接支持css的处理 直接导入css即可； **
  • vite可以直接支持css预处理器，比如less ,sass
    • 直接导入less,安装less编译器 npm i less -D
  • **vite直接支持postcss的转换： **
    • 只需要安装postcss，并且配置 postcss.config.js 的配置文件即可；
    • npm install postcss postcss-preset-env -D

Vite对TypeScript是原生支持的，它会直接使用ESBuild来完成编译：只需要直接导入即可 , 如果我们查看浏览器中的请求，会发现请求的依然是ts的代码：这是因为vite中的服务器Connect会对我们的请求进行转发；获取ts编译后的代码，给浏览器返回，浏览器可以直接进行解析

  • 注意：在vite2中，已经不再使用Koa了，而是使用Connect来搭建的服务器
    • 由于大多数逻辑应该通过插件钩子而不是中间件来完成，因此对中间件的需求大大减少。内部服务器应用现在是一个很好的旧版的 connect 实例，而不是 Koa。

在开发中，我们不可能所有的项目都使用vite从零去搭建，比如一个react项目、Vue项目；

这个时候vite还给我们提供了对应的脚手架工具,Vite实际上是有两个工具的：

  • vite：相当于是一个构件工具，类似于webpack、rollup；
  • @vitejs/create-app：类似vue-cli、create-react-app；

如果使用脚手架工具呢？

ESBuild特点

  • 超快的构建速度，并且不需要缓存;
  • 支持ES6和CommonJS的模块化;
  • 支持ES6的Tree Shaking;
  • 支持Go、JavaScript的APl;支持TypeScript、JSX等语法编译;支持SourceMap;
  • 支持代码压缩;
  • 支持扩展其他插件;

ESBuild为什么这么快呢?

  • 使用Go语言编写的，可以直接转换成机器代码，而无需经过字节码;
  • ESBuild可以充分利用CPU的多内核，尽可能让它们饱和运行;
  • ESBuild的所有内容都是从零开始编写的，而不是使用第三方，所以从一开始就可以考虑各种性能问题;
  • 等等…
