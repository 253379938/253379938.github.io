Ts 类型检测-鸭子类型

在 TypeScript 里，鸭子类型（Duck Typing）也叫结构子类型（Structural Subtyping），它的核心观念是：若一个对象满足了某个类型的结构特征，那么它就可被当作该类型来使用

鸭子类型关注对象的结构而不是其具体的类型

映射类型( Mapped Types )

在 TypeScript 里，映射能依据现有类型来创建新类型。借助类型映射，你能够对已有类型中的每个属性进行转换，进而生成新类型。

基本语法

  • T 是要映射的原始类型。
  • Property 是一个类型变量，代表 T 中的每个属性名。
  • keyof Type 是一个索引类型查询操作符，它返回 Type 中所有属性名组成的联合类型。
  • T[Property] 是映射后每个属性的新类型。此处是原值 , 可自定义为新值

修饰符

类型声明文件和内置类型声明

TypeScript 类型声明文件（.d.ts 文件）的作用是为 JavaScript 代码提供类型信息，让 TypeScript 编译器能够理解和检查 JavaScript 代码

声明文件以 .d.ts 为扩展名，它不会生成 JavaScript 代码，仅用于提供类型信息。例如，当你使用第三方 JavaScript 库时，就可以通过类型声明文件为这个库添加类型支持。

  • 内置类型声明
    • 内置类型声明是 typescript 提供的、帮助我们内置了 JavaScript 运行时的一些标准化 API 的声明文件
    • 包括比如 Function、String、Math、Date 等内置类型;
    • 运行环境中的 DOM API，比如 Window、Document 等;
  • 外部定义类型声明( 第三方库 )
    • 在自己库中存在类型声明(编写.d.ts 文件)，比如 axios
    • 通过社区的一个公有库 DefinitelyTyped 存放类型声明文件
  • 自定义类型声明

tsconfig.json配置

  • **target**：指定编译后 JavaScript 代码的 ECMAScript 版本，比如 ES3、ES5、ES6 等
  • **module**：指定生成代码的模块系统，可选值有 commonjs、amd、system、umd、es6 等
  • **lib**：指定要包含在编译中的库文件，例如 es5、es6、dom 等
  • **strict**：启用所有严格类型检查选项，开启后能提高代码的类型安全性
  • **esModuleInterop**：允许在 CommonJS 模块中使用 ES6 模块的导入语法
  • **skipLibCheck**：跳过对所有库类型检查( 仅检测你所用的类型 )
  • **forceConsistentCasingInFileNames**：确保文件名的大小写一致，避免在不同操作系统上出现问题

条件类型

在 TypeScript 里，条件类型可以依据条件来选择不同的类型。它的语法和 JavaScript 里的三元运算符很相似，格式为 T extends U ? X : Y，意思是若 T 能够赋值给 U，那么类型就是 X，反之则是 Y。

  • ReturnType 获取返回值的类型

infer 关键字可以在条件类型里推断类型

GetReturnType 条件类型接收一个函数类型 T。通过 infer R 推断出函数的返回类型

当泛型参数为联合类型，并且在条件类型里直接使用该泛型参数时，条件类型会自动对联合类型的每个成员进行分发

内置工具

Partial : 所有属性转为可选

Required : 所有属性转为必选

Readonly : 所有属性转为只读

**Pick<T, K>**

作用：从类型 T 中选取指定属性 K 组成新类型。
实现：通过 keyof 提取 T 的所有键，K extends keyof T 约束 K 必须是 T 的键，最后通过映射类型 [P in K]: T[P] 生成新对象

**Omit<T, K>**

作用：从类型 T 中剔除指定属性 K。
实现：结合 Exclude 和 Pick，先用 Exclude<keyof T, K> 获取未被剔除的键，再用 Pick 选取

**Exclude<T, U>**** 和 ****Extract<T, U>**

作用：Exclude 从联合类型 T 中排除 U，Extract 保留 T 中与 U 重叠的部分。
实现：基于条件类型 T extends U ? never : T 和 T extends U ? T : never。

  • NonNullable : 去除 null 和 undefined 类型
