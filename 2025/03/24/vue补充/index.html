自定义指令

自定义指令是 Vue3 中用于封装底层 DOM 操作的高级特性，适用于需要直接操作元素样式、事件或生命周期的场景。

  • 逻辑复用：将特定 DOM 行为抽象为可复用的指令
  • 关注点分离：避免组件中混杂复杂的 DOM 操作逻辑
  • 性能优化：减少不必要的组件渲染开销

注意：仅在必须直接操作 DOM 时使用指令，优先考虑组件化方案。避免在多根节点组件上使用指令，可能引发不可预期的行为。

自定义指令生命周期

Vue3 的指令生命周期与组件生命周期高度对齐，提供七个关键钩子

钩子名称 触发时机
created
beforeMount 指令首次绑定元素且父组件挂载前
mounted 父组件挂载完成，元素已插入 DOM（最常用钩子）
beforeUpdate 组件更新前，元素对应的 VNode 更新前
updated 组件更新后，元素及其子节点的 VNode 更新完成
beforeUnmount 父组件卸载前，指令与元素解绑前
unmounted 指令与元素完全解绑，父组件卸载完成（清理资源的最佳时机）
自定义指令参数修饰符
  • el：指令绑定的 DOM 元素。
  • binding：包含以下属性的对象：
    • value：指令绑定的值（如 v-my-directive="42"，则 value 为 42）。
    • oldValue：更新前的值（仅在 beforeUpdate 和 updated 中可用）。
    • arg：指令参数（如 v-my-directive:foo，则 arg 为 'foo'）。
    • modifiers：修饰符对象（如 v-my-directive.modif，则 modifiers 为 { modif: true }）。
    • instance：当前组件实例
  • vnode：虚拟节点。
  • prevNode：上一个虚拟节点（仅在 beforeUpdate 和 updated 中可用）。
示例
  1. 优先使用简写形式：若 mounted 和 updated 行为一致，可简化为函数形式。
  2. 资源清理：在 beforeUnmount 中移除事件监听器，避免内存泄漏。
  3. 避免组件依赖：尽量通过 binding.value 传递参数，减少对 instance 的依赖。

Suspense

Suspense 是 Vue 3 提供的一个内置组件，主要用于处理异步依赖的加载状态。在开发中，当组件依赖一些异步操作（比如异步组件、异步数据获取）时，在数据加载完成之前显示一个加载中的状态，加载完成后再显示实际内容。( 实验性功能 )

  • <Suspense> 组件有两个插槽：
    • #default：当异步依赖加载完成后，显示这个插槽里的内容。
    • #fallback：在异步依赖加载过程中，显示这个插槽里的内容，一般用于展示加载中的提示信息。

vue插件本质

在 Vue 里，插件本质上是一种扩展 Vue 功能的方式，它可以封装一些通用逻辑，从而在多个项目或者组件中复用。安装插件的本质是借助 app.use() 方法把插件集成到 Vue 应用实例里，让插件的功能在整个应用中生效。

Vue 插件一般是一个对象，该对象包含 install 方法；或者是一个函数，这个函数本身就作为 install 方法。

基本用法

vue过渡动画

Vue 3 提供了强大的过渡动画系统，主要通过 <Transition> 和 <TransitionGroup> 组件实现元素的进入/离开动画，以及列表动画

**<Transition>**** 组件**

  • 作用：为单个元素或组件添加进入/离开过渡动画。
  • 核心机制：自动在元素进入/离开时添加特定的 CSS 类名，触发 CSS 过渡或动画。

CSS 过渡类名

  • 进入阶段：
    • v-enter-from: 进入动画的起始状态。
    • v-enter-active: 进入动画的持续时间和缓动函数。
    • v-enter-to: 进入动画的结束状态。
  • 离开阶段：
    • v-leave-from: 离开动画的起始状态。
    • v-leave-active: 离开动画的持续时间和缓动函数。
    • v-leave-to: 离开动画的结束状态。
  • 自定义类名前缀：通过 name 属性修改前缀。例如，name="fade" 会生成 fade-enter-from 等类名。

使用 <TransitionGroup> 组件为列表元素添加过渡动画。<TransitionGroup> 用于多个元素或组件的过渡效果，它会在列表中的元素插入、删除或移动时应用过渡动画

  • **key**** **属性：在 <TransitionGroup> 中，每个子元素都必须有唯一的 key，这样 Vue 才能正确识别元素的插入、删除和移动。
  • **tag**** 属性**：tag 属性用于指定 <TransitionGroup> 渲染的根元素，如果不指定，默认会渲染为一个 <span> 元素。
  • 移动动画：除了插入和删除动画，<TransitionGroup> 还支持元素移动时的动画效果。可以通过设置 move-class 属性来定义移动动画的类名。
  • 定义离开动画时,尽量使用position: absolute 作用是确保元素在离开时不会影响其他元素的布局，从而正确执行过渡动画
    • 不加 position: absolute
      • 元素仍处于文档流中。
      • 其他元素会立即占据被移除元素的位置。或者删除时,仍占据位置直至从 DOM 中移除导致其他元素因被移除元素的位置被占据无法过渡动画
    • 加 position: absolute
      • 元素脱离文档流，其他元素不会移动。
      • 元素在原位执行过渡动画（如向右移动），动画结束后再从 DOM 中移除。

响应式

响应式本质

Vue 的响应式系统是其核心机制之一，它通过数据劫持和依赖追踪实现数据变化到视图更新的自动化

响应式系统的本质：当数据变化时，自动触发依赖该数据的代码（如视图渲染、计算属性等），实现数据和 UI 的同步。

vue2响应式

Vue 2 响应式系统的核心逻辑，主要借助 Object.defineProperty 方法对对象属性进行劫持，以此实现数据的响应式更新。通过发布 - 订阅模式，当对象的属性值发生变化时，能够自动触发与之关联的依赖函数。

reactive函数实现属性劫持与依赖收集

  • 该函数接收一个对象 obj 作为参数，通过 Object.keys(obj).forEach 遍历对象的所有属性。
  • 对于每个属性，使用 Object.defineProperty 进行劫持：
    • **setter**：当属性值被修改时，更新内部的 value 变量，然后通过 getDep 函数获取该属性对应的 Depend 对象，调用其 notify 方法通知所有依赖函数执行。
    • **getter**：在属性被访问时，通过 getDep 函数获取该属性对应的 Depend 对象，调用其 depent 方法将当前的 reactiveFn 收集为依赖，最后返回属性的内部值 value。
  • 最后返回经过劫持处理后的对象 obj。

**Depend**** 类**

  • Depend 类用于管理依赖函数，内部使用 Set 类型的 watchFnList 存储依赖函数，以避免同一函数被重复添加。
    • **addDepend**** **方法：将传入的依赖函数 fn 添加到 watchFnList 中。
    • **depent**** **方法：如果 reactiveFn 存在，则将其添加到 watchFnList 中。
    • **notify**** 方法**：遍历 watchFnList，依次执行其中的每个依赖函数。

存储依赖信息映射

  • objMap 是一个 WeakMap，用于存储对象和其对应的依赖信息映射。
  • getDep 函数用于根据对象 obj 和属性名 key 获取对应的 Depend 对象：
    • 首先从 objMap 中查找对象 obj 对应的 Map。
    • 如果该 Map 不存在，则创建一个新的 Map 并将其存入 objMap。
    • 然后从这个 Map 中查找属性 key 对应的 Depend 对象。
    • 如果该 Depend 对象不存在，则创建一个新的 Depend 对象并将其存入 Map。
    • 最后返回找到或创建的 Depend 对象。

**watchFn**** 函数**

  • reactiveFn 是一个全局变量，用于临时存储当前正在执行的依赖函数。
  • watchFn 函数接收一个依赖函数 fn 作为参数：
    • 将 fn 赋值给 reactiveFn。
    • 执行 fn，在执行过程中，如果访问了被劫持的属性，会触发该属性的 getter，从而将 reactiveFn 收集为依赖。
    • 执行完 fn 后，将 reactiveFn 置为 null。

在vue中模板语法最后都会转成Vnode , 最后被渲染函数h函数去渲染 , h函数中就使用了类似watchFn的函数 , 实现可以依赖响应式数据实现数据响应式

总结

  • 响应式原理：利用 Object.defineProperty 对对象属性进行劫持，在 getter 中收集依赖，在 setter 中触发依赖更新。
  • 依赖管理：使用 Depend 类和 Set 数据结构管理依赖函数，避免重复添加。
  • 多对象支持：通过 WeakMap 和 Map 存储不同对象及其属性对应的依赖信息，支持对多个对象的属性进行劫持和依赖管理。
  • 使用方式：通过 reactive 函数将普通对象转换为响应式对象，使用 watchFn 函数包裹依赖函数，实现依赖的自动收集和响应式更新。
vue3响应式

Vue 3 响应式系统，能够对多个对象的属性变化进行监听，并自动收集依赖。当对象的属性值发生改变时，与之关联的依赖函数会被自动触发执行

**reactive**** 函数数据拦截和依赖收集**

  • 该函数接收一个普通对象 obj 作为参数，使用 Proxy 对象对其进行代理。
    • **set**** **拦截器：当修改代理对象的属性时，此拦截器会被触发。借助 Reflect.set 方法更新目标对象 target 上的属性值。然后通过 getDep 函数获取该属性对应的 Depend 对象，调用其 notify 方法通知所有依赖函数执行。
    • **get**** 拦截器**：当访问代理对象的属性时，该拦截器会被触发。通过 getDep 函数获取该属性对应的 Depend 对象，调用其 depent 方法将当前的 reactiveFn 收集为依赖，最后借助 Reflect.get 方法返回目标对象 target 上该属性的值。
  • 最终返回代理对象 objProxy。
总结
特性 Vue 2 (Object.defineProperty) Vue 3 (Proxy)
对象属性新增/删除 不支持，需用 Vue.set
/Vue.delete		 原生支持
数组索引修改 需重写方法或 Vue.set 原生支持
初始化性能 递归遍历所有属性，性能较差 惰性代理，按需响应式
代码复杂度 实现简单 依赖复杂的数据结构（WeakMap/Map）

响应式工作流程

  1. 数据劫持：通过 Object.defineProperty 或 Proxy 监听数据读写。
  2. 依赖收集：在 getter 中收集当前正在执行的依赖（如组件的渲染函数）。
  3. 触发更新：在 setter 中通知所有依赖执行更新逻辑（如重新渲染组件）。
