认识Vue

Vue 是一款用于构建用户界面的 JavaScript 框架。它基于标准 HTML、CSS 和 JavaScript 构建，并提供了一套声明式的、组件化的编程模型 。

  • 无需构建步骤，渐进式增强静态的 HTML
  • 在任何页面中作为 Web Components 嵌入
  • 单页应用 (SPA)
  • 全栈 / 服务端渲染 (SSR)
  • Jamstack / 静态站点生成 (SSG)
  • 开发桌面端、移动端、WebGL，甚至是命令行终端中的界面

MVC vs MVVM
  1. MVC（Model-View-Controller）
    • 组成：
      • Model：数据逻辑（如 API 请求、数据库操作）。
      • View：用户界面（如 HTML/CSS）。
      • Controller：接收用户输入，协调 Model 和 View。
    • 数据流：
      用户操作 → Controller → 更新 Model → View 从 Model 拉取数据渲染。
    • 痛点：View 和 Model 可能紧密耦合，需手动同步。
  2. MVVM（Model-View-ViewModel）
    • 组成：
      • Model：数据层（同 MVC）。
      • View：界面（模板）。
      • ViewModel：数据绑定与同步（Vue 实例）。
    • 数据流：
      • 双向绑定：View 中表单输入自动更新 Model（如 v-model）。
      • 响应式更新：Model 变化自动触发 View 刷新（通过数据劫持）。

data

  • 作用：用于定义组件的数据。它是一个函数，返回一个包含数据的对象。
  • data属性是传入一个函数，并且该函数需要返回一个对象

  • 原理:
    • data中返回的对象会被Vue的响应式系统劫持, 之后对该对象的修改或访问都会在劫持中被处理 ,所以我们在template或者app中通过 访问counter，可以从对象中获取到数据. 所以我们修改counter的值时，app中的 也会发生改变

  • 作用：用于定义组件的方法。这些方法可以在模板中通过事件绑定调用，也可以在组件的其他方法中调用。
  • methods属性是一个对象，通常我们会在这个对象中定义很多的方法:

computed

在 Vue 中，计算属性（Computed） 是一种基于响应式依赖进行缓存的属性，用于简化模板中的复杂逻辑，避免重复计算

定义

  • 计算属性是声明式的属性，通过依赖其他响应式数据动态计算得出值。
  • 当依赖的数据变化时，计算属性会自动重新计算，否则直接返回缓存结果

语法

  • 选项式API(Vue2/Vue3)
  • 组合式API(Vue3)

核心特点

1. 缓存机制

  • 缓存结果：只有依赖的响应式数据变化时才会重新计算。
  • 对比方法（Methods）：方法每次调用都会执行逻辑。

2. 响应式依赖

  • 自动追踪依赖：计算属性内部使用的所有响应式数据都会被追踪。
  • 非响应式数据不会触发更新（如 Date.now()）。

3. 同步计算

  • 不支持异步：计算属性必须是同步操作（异步需用 watch 或方法）。

4. 可写计算属性（Setter）

key

  • 必要性：为每个元素绑定唯一标识，帮助 Vue 高效更新 DOM。
  • 正确用法：使用稳定唯一值（如 id），避免索引或随机数。
  • 官方解释
    • key属性主要用在Vue的虚拟DOM算法，在新旧nodes对比时辨识VNodes
    • 如果不使用key，Vue会使用一种最大限度减少动态元素并且尽可能的尝试就地修改/复用相同类型元素的算法
    • 而使用key时，它会基于key的变化重新排列元素顺序，并且会移除/销毁key不存在的元素
  • 原理
    • Vue 的虚拟 DOM 是一种轻量级的 JavaScript 对象，它是真实 DOM 的抽象表示。在更新 DOM 时，Vue 会通过比较新旧虚拟 DOM 的差异( VNode与真实Node的区别 )，只更新需要更新的真实 DOM 节点，以提高性能。key 就是用来帮助 Vue 识别哪些元素发生了变化，从而更高效地进行 DOM 更新。
      • 虚拟 DOM 可以做跨平台
      • 虚拟 DOM 做Diff算法
    • https://cn.vuejs.org/guide/extras/rendering-mechanism.html#virtual-dom

Diff算法-Vue

Diff 算法是一种用于比较两个虚拟 DOM 树差异的算法，它的目标是找出最少的 DOM 操作来更新真实 DOM，从而提高性能。

Vue 的 Diff 算法采用了双指针法和同层比较的策略。它只比较同一层级的节点，不会跨层级比较，这样可以将时间复杂度从指数级降低到线性级（(O(n))）。Diff 算法会对新旧虚拟 DOM 树的同一层级节点进行比较，根据节点的不同情况进行相应的处理，如创建新节点、更新节点属性、移动节点或删除节点等。

在比较同一层级的节点时，会根据节点的 key、标签名、属性等信息进行判断

  • 如果新旧节点的标签名不同，直接用新节点替换旧节点
  • 如果新旧节点的标签名相同，但 key 不同，也会用新节点替换旧节点
  • 如果新旧节点的标签名和 key 都相同，则会复用旧节点，只更新节点的属性和内容。

当比较列表节点时，会使用 key 来提高比较效率。key 可以帮助 Vue 准确地识别哪些节点是新增的、哪些节点是需要移除的、哪些节点只是位置发生了变化。

浏览器内模板编译注意事项

当以无构建步骤方式使用 Vue 时，组件模板要么是写在页面的 HTML 中，要么是内联的 JavaScript 字符串。在这些场景中，为了执行动态模板编译，Vue 需要将模板编译器运行在浏览器中。相对的，如果我们使用了构建步骤，由于提前编译了模板，那么就无须再在浏览器中运行了。为了减小打包出的客户端代码体积，Vue 提供了多种格式的“构建文件”以适配不同场景下的优化需求。

  • 前缀为 vue.runtime.* 的文件是只包含运行时的版本：不包含编译器，当使用这个版本时，所有的模板都必须由构建步骤预先编译。
  • 名称中不包含 .runtime 的文件则是完全版：即包含了编译器，并支持在浏览器中直接编译模板。然而，体积也会因此增长大约 14kb。

默认的工具链中都会使用仅含运行时的版本，因为所有单文件组件中的模板都已经被预编译了。如果因为某些原因，在有构建步骤时，你仍需要浏览器内的模板编译，你可以更改构建工具配置，将 vue 改为相应的版本 vue/dist/vue.esm-bundler.js。
简单来说使用vue文件,vue-loader会预先进行编译compile,所以可以引入更轻量的vue,如果使用对象形式,那么就需要使用vue/dist/vue.esm-bundler.js里面的compile进行编译vue文件

插槽

插槽（Slot）是一种非常强大且实用的特性，它允许你在组件中灵活地插入内容，实现组件的复用和定制。

默认插槽 (匿名插槽)

默认插槽是最基本的插槽类型，当父组件向子组件传递内容时，如果没有指定插槽名称，内容将被插入到默认插槽中。

具名插槽

具名插槽允许你在一个组件中定义多个插槽，并通过名称来区分它们。父组件可以根据插槽名称将内容插入到指定的插槽中。

  • 默认插槽的name 属性为 default
  • 在子组件中，通过 name 属性为 <slot> 标签指定名称，定义了具名插槽。
  • 在父组件中，使用 <template> 标签并通过 # 语法（v-slot 的缩写）指定插槽名称，将内容插入到对应的插槽中。未指定名称的内容将被插入到默认插槽中。

动态插槽名

动态插槽名，允许你在运行时动态决定要使用的插槽名称。

  • 在父组件中，使用 v-slot:[dynamicSlotName] 语法，dynamicSlotName 是一个动态变量，它的值决定了要使用的插槽名称。

作用域插槽

作用域插槽允许子组件向父组件传递数据，父组件可以在插入内容时使用这些数据。

  • 在子组件中，通过 : 语法将 items 数据绑定到 <slot> 标签上，向父组件传递数据。
  • 在父组件中，使用 <template> 标签并通过 #default 指定默认插槽，同时使用解构赋值接收子组件传递的数据，然后在插入内容时使用这些数据。

  • 适用场景：祖先向后代传递数据（非响应式）

生命周期

Vue 组件的生命周期是理解组件行为的关键，它描述了组件从创建到销毁的整个过程。每个生命周期阶段都提供了特定的钩子函数（Hooks），允许开发者在不同时机执行自定义逻辑。

Vue 生命周期流程

  1. 初始化阶段
    beforeCreate → created
  2. 挂载阶段
    beforeMount → mounted
  3. 更新阶段
    beforeUpdate → updated
  4. 销毁阶段
    beforeDestroy → destroyed

初始化阶段

  • **beforeCreate**
    • 触发时机：实例初始化之后，数据观测（data）和事件配置（methods）之前。
    • 特点：
      • 无法访问 data、computed、methods 等属性。
      • 常用于插件初始化（如 Vuex、Vue Router）。
  • **created**
    • 触发时机：实例创建完成，数据观测和事件配置已初始化。
    • 特点：
      • 可以访问 data、methods，但 DOM 未生成（无法操作 DOM）。
      • 适合发起异步请求、初始化数据。

2. 挂载阶段

  • **beforeMount**
    • 触发时机：模板编译完成，但尚未挂载到 DOM。
    • 特点：
      • 虚拟 DOM 已生成，但页面还未渲染。
  • **mounted**
    • 触发时机：实例挂载到 DOM 后触发。
    • 特点：
      • 可以操作 DOM，适合初始化第三方库（如图表、地图）。
      • 注意：子组件的 mounted 可能未完成，需用 this.$nextTick 确保全部渲染完成。

3. 更新阶段

  • **beforeUpdate**
    • 触发时机：数据变化导致虚拟 DOM 重新渲染前。
    • 特点：
      • 数据已更新，但 DOM 尚未重新渲染。
      • 可在此处获取更新前的 DOM 状态。
  • **updated**
    • 触发时机：虚拟 DOM 重新渲染并应用到 DOM 后。
    • 特点：
      • 避免在此钩子中修改数据，可能导致无限循环。
      • 需要操作更新后的 DOM 时，建议使用 this.$nextTick。

4. 销毁阶段

  • **beforeDestroy**
    • 触发时机：实例销毁前。
    • 特点：
      • 组件仍完全可用。
      • 适合清理定时器、解绑事件、取消网络请求等资源释放操作。
  • **destroyed**
    • 触发时机：实例销毁后。
    • 特点：
      • 所有子实例和事件监听器已被移除。

Vue 3 的变化

  1. 组合式 API 的钩子
    • 钩子名称前加 on，如 onMounted、onUpdated。
    • beforeCreate 和 created 被 setup() 替代。
  2. 新增钩子
    • onRenderTracked / onRenderTriggered：用于调试渲染依赖。
