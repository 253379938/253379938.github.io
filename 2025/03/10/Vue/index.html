认识Vue

Vue 是一款用于构建用户界面的 JavaScript 框架。它基于标准 HTML、CSS 和 JavaScript 构建，并提供了一套声明式的、组件化的编程模型 。

  • 无需构建步骤，渐进式增强静态的 HTML
  • 在任何页面中作为 Web Components 嵌入
  • 单页应用 (SPA)
  • 全栈 / 服务端渲染 (SSR)
  • Jamstack / 静态站点生成 (SSG)
  • 开发桌面端、移动端、WebGL，甚至是命令行终端中的界面

MVC vs MVVM
  1. MVC（Model-View-Controller）
    • 组成：
      • Model：数据逻辑（如 API 请求、数据库操作）。
      • View：用户界面（如 HTML/CSS）。
      • Controller：接收用户输入，协调 Model 和 View。
    • 数据流：
      用户操作 → Controller → 更新 Model → View 从 Model 拉取数据渲染。
    • 痛点：View 和 Model 可能紧密耦合，需手动同步。
  2. MVVM（Model-View-ViewModel）
    • 组成：
      • Model：数据层（同 MVC）。
      • View：界面（模板）。
      • ViewModel：数据绑定与同步（Vue 实例）。
    • 数据流：
      • 双向绑定：View 中表单输入自动更新 Model（如 v-model）。
      • 响应式更新：Model 变化自动触发 View 刷新（通过数据劫持）。

data

  • 作用：用于定义组件的数据。它是一个函数，返回一个包含数据的对象。
  • data属性是传入一个函数，并且该函数需要返回一个对象

computed

在 Vue 中，计算属性（Computed） 是一种基于响应式依赖进行缓存的属性，用于简化模板中的复杂逻辑，避免重复计算

定义

  • 计算属性是声明式的属性，通过依赖其他响应式数据动态计算得出值。
  • 当依赖的数据变化时，计算属性会自动重新计算，否则直接返回缓存结果

语法

  • 选项式API(Vue2/Vue3)
  • 组合式API(Vue3)

核心特点

**1. 缓存机制 **

  • 缓存结果：只有依赖的响应式数据变化时才会重新计算。
  • 对比方法（Methods）：方法每次调用都会执行逻辑。

2. 响应式依赖

  • 自动追踪依赖：计算属性内部使用的所有响应式数据都会被追踪。
  • 非响应式数据不会触发更新（如 Date.now()）。

3. 同步计算

  • 不支持异步：计算属性必须是同步操作（异步需用 watch 或方法）。

4. 可写计算属性（Setter）

key

  • 必要性：为每个元素绑定唯一标识，帮助 Vue 高效更新 DOM。
  • 正确用法：使用稳定唯一值（如 id），避免索引或随机数。
  • 官方解释
    • key属性主要用在Vue的虚拟DOM算法，在新旧nodes对比时辨识VNodes
    • 如果不使用key，Vue会使用一种最大限度减少动态元素并且尽可能的尝试就地修改/复用相同类型元素的算法
    • 而使用key时，它会基于key的变化重新排列元素顺序，并且会移除/销毁key不存在的元素
  • 原理
    • Vue 的虚拟 DOM 是一种轻量级的 JavaScript 对象，它是真实 DOM 的抽象表示。在更新 DOM 时，Vue 会通过比较新旧虚拟 DOM 的差异( VNode与真实Node的区别 )，只更新需要更新的真实 DOM 节点，以提高性能。key 就是用来帮助 Vue 识别哪些元素发生了变化，从而更高效地进行 DOM 更新。
      • 虚拟 DOM 可以做跨平台
      • 虚拟 DOM 做Diff算法
    • https://cn.vuejs.org/guide/extras/rendering-mechanism.html#virtual-dom

Diff算法-Vue

Diff 算法是一种用于比较两个虚拟 DOM 树差异的算法，它的目标是找出最少的 DOM 操作来更新真实 DOM，从而提高性能。

Vue 的 Diff 算法采用了双指针法和同层比较的策略。它只比较同一层级的节点，不会跨层级比较，这样可以将时间复杂度从指数级降低到线性级（(O(n))）。Diff 算法会对新旧虚拟 DOM 树的同一层级节点进行比较，根据节点的不同情况进行相应的处理，如创建新节点、更新节点属性、移动节点或删除节点等。

在比较同一层级的节点时，会根据节点的 key、标签名、属性等信息进行判断

  • 如果新旧节点的标签名不同，直接用新节点替换旧节点
  • 如果新旧节点的标签名相同，但 key 不同，也会用新节点替换旧节点
  • 如果新旧节点的标签名和 key 都相同，则会复用旧节点，只更新节点的属性和内容。

当比较列表节点时，会使用 key 来提高比较效率。key 可以帮助 Vue 准确地识别哪些节点是新增的、哪些节点是需要移除的、哪些节点只是位置发生了变化。

浏览器内模板编译注意事项

当以无构建步骤方式使用 Vue 时，组件模板要么是写在页面的 HTML 中，要么是内联的 JavaScript 字符串。在这些场景中，为了执行动态模板编译，Vue 需要将模板编译器运行在浏览器中。相对的，如果我们使用了构建步骤，由于提前编译了模板，那么就无须再在浏览器中运行了。为了减小打包出的客户端代码体积，Vue 提供了多种格式的“构建文件”以适配不同场景下的优化需求。

  • 前缀为 vue.runtime.* 的文件是只包含运行时的版本：不包含编译器，当使用这个版本时，所有的模板都必须由构建步骤预先编译。
  • 名称中不包含 .runtime 的文件则是完全版：即包含了编译器，并支持在浏览器中直接编译模板。然而，体积也会因此增长大约 14kb。

默认的工具链中都会使用仅含运行时的版本，因为所有单文件组件中的模板都已经被预编译了。如果因为某些原因，在有构建步骤时，你仍需要浏览器内的模板编译，你可以更改构建工具配置，将 vue 改为相应的版本 vue/dist/vue.esm-bundler.js。
简单来说使用vue文件,vue-loader会预先进行编译compile,所以可以引入更轻量的vue,如果使用对象形式,那么就需要使用vue/dist/vue.esm-bundler.js里面的compile进行编译vue文件

非Prop的Attribute (了解)

  • 当我们传递给一个组件某个属性，但是该属性并没有定义对应的props或者emits时，就称之为 非Prop的Attribute

插槽

插槽（Slot）是一种非常强大且实用的特性，它允许你在组件中灵活地插入内容，实现组件的复用和定制。

默认插槽 (匿名插槽)

默认插槽是最基本的插槽类型，当父组件向子组件传递内容时，如果没有指定插槽名称，内容将被插入到默认插槽中。

具名插槽

具名插槽允许你在一个组件中定义多个插槽，并通过名称来区分它们。父组件可以根据插槽名称将内容插入到指定的插槽中。

  • 默认插槽的name 属性为 default
  • 在子组件中，通过 name 属性为 <slot> 标签指定名称，定义了具名插槽。
  • 在父组件中，使用 <template> 标签并通过 # 语法（v-slot 的缩写）指定插槽名称，将内容插入到对应的插槽中。未指定名称的内容将被插入到默认插槽中。

生命周期

Vue 组件的生命周期是理解组件行为的关键，它描述了组件从创建到销毁的整个过程。每个生命周期阶段都提供了特定的钩子函数（Hooks），允许开发者在不同时机执行自定义逻辑。

Vue 生命周期流程

  1. 初始化阶段
    beforeCreate → created
  2. 挂载阶段
    beforeMount → mounted
  3. 更新阶段
    beforeUpdate → updated
  4. 销毁阶段
    beforeDestroy → destroyed

初始化阶段

  • **beforeCreate**
    • 触发时机：实例初始化之后，数据观测（data）和事件配置（methods）之前。
    • 特点：
      • 无法访问 data、computed、methods 等属性。
      • 常用于插件初始化（如 Vuex、Vue Router）。
  • **created**
    • 触发时机：实例创建完成，数据观测和事件配置已初始化。
    • 特点：
      • 可以访问 data、methods，但 DOM 未生成（无法操作 DOM）。
      • 适合发起异步请求、初始化数据。

2. 挂载阶段

  • **beforeMount**
    • 触发时机：模板编译完成，但尚未挂载到 DOM。
    • 特点：
      • 虚拟 DOM 已生成，但页面还未渲染。
  • **mounted**
    • 触发时机：实例挂载到 DOM 后触发。
    • 特点：
      • 可以操作 DOM，适合初始化第三方库（如图表、地图）。
      • 注意：子组件的 mounted 可能未完成，需用 this.$nextTick 确保全部渲染完成。

3. 更新阶段

  • **beforeUpdate**
    • 触发时机：数据变化导致虚拟 DOM 重新渲染前。
    • 特点：
      • 数据已更新，但 DOM 尚未重新渲染。
      • 可在此处获取更新前的 DOM 状态。
  • **updated**
    • 触发时机：虚拟 DOM 重新渲染并应用到 DOM 后。
    • 特点：
      • 避免在此钩子中修改数据，可能导致无限循环。
      • 需要操作更新后的 DOM 时，建议使用 this.$nextTick。

4. 销毁阶段

  • **beforeDestroy**
    • 触发时机：实例销毁前。
    • 特点：
      • 组件仍完全可用。
      • 适合清理定时器、解绑事件、取消网络请求等资源释放操作。
  • **destroyed**
    • 触发时机：实例销毁后。
    • 特点：
      • 所有子实例和事件监听器已被移除。

混入Mixin

混入（mixin）提供了一种灵活的方式，来分发组件中的可复用功能。一个混入对象可以包含任意组件选项，例如 data、methods、computed 等。当一个组件使用混入对象时，混入对象的选项会被合并到组件本身的选项中。

全局混入

全局混入会影响每一个之后创建的 Vue 实例。使用 Vue.mixin() 来全局注册混入对象。

局部混入

局部混入只会影响使用该混入对象的组件。在组件中使用 mixins 选项来引入混入对象。

当组件和混入对象包含同名选项时，会根据选项类型进行不同的合并规则：

  • 如果数据对象在组件和混入对象中都存在，它们会进行递归合并，并且组件的数据会覆盖混入对象的数据
  • 同名的钩子函数会被合并成一个数组，因此都会被调用。混入对象的钩子函数会在组件自身钩子函数之前调用。
  • 值为对象的选项，如 methods、computed、watch 等，会进行合并。如果键名冲突，组件的选项会覆盖混入对象的选项。

组合式 API (Composition API)

Vue 3 引入了组合式 API（Composition API），它是一种新的组织和复用代码的方式，相较于选项式 API 提供了更灵活、更高效的代码组织方式。

组合式 API 允许你使用函数来组织逻辑，而不是像选项式 API 那样将逻辑分散在不同的选项（如 data、methods、computed 等）中。主要通过 setup 函数来使用组合式 API，setup 函数在组件初始化时执行，在 data 和 props 被解析之后，beforeCreate 钩子之前调用。

setup函数

**setup**** 函数**

setup函数是一个新的组件选项，它在组件初始化时，props解析之后，data和computed等选项之前执行。该函数接收两个参数：props和context，并且可以返回一个对象，对象中的属性和方法可以在模板中使用。

  • 注意：setup 内无法访问 this

props是一个响应式对象，包含了父组件传递给子组件的所有属性。需要注意的是，不能使用 ES6 解构，因为解构会破坏props的响应式。

context是一个普通对象，包含了三个属性：attrs、slots和emit。

  • attrs：包含了所有未在props选项中声明的属性。
  • slots：用于访问组件的插槽内容。
  • emit：用于触发自定义事件。

setup函数可以返回一个对象，对象中的属性和方法可以在模板中直接使用。此外，setup函数还可以返回一个渲染函数，用于手动创建虚拟 DOM。

  • **this**的使用：在setup函数中，this不会指向当前组件实例，因此不能使用this来访问组件的属性和方法。
  • 执行时机：setup函数在组件实例初始化之前执行，因此不能在setup函数中访问data、computed等选项。
  • 生命周期钩子：如果需要使用生命周期钩子，可以在setup函数中使用组合式 API 提供的生命周期函数，如onMounted、onUpdated等。

响应式数据

reactive 函数

reactive 函数用于创建一个响应式的对象或数组。它基于 ES6 的 Proxy 实现，能够深度监听对象属性的变化。

  • 只能用于对象和数组：reactive 只能处理对象和数组，不能处理基本数据类型（如 number、string、boolean 等）。
  • 深层响应式：reactive 会递归地将对象的所有属性转换为响应式的，因此可以深度监听对象的变化。
ref 函数

ref 函数用于创建一个响应式的引用，它可以处理基本数据类型和对象。ref 创建的对象有一个 .value 属性，用于访问和修改其内部的值。

  • 在模板中自动解包：在模板中使用 ref 对象时，不需要手动访问 .value 属性，Vue 会自动解包。但在 JavaScript 代码中，需要使用 .value 来访问和修改其值。
  • 可以处理基本数据类型：ref 可以处理基本数据类型，这是它与 reactive 的主要区别之一。
响应式原理

**Vue 3 使用 Proxy 替代 Vue 2 的 ****Object.defineProperty**，解决了以下问题：

  1. 对数组的深度监听：无需重写数组方法（如 push, pop）。
  2. 动态新增/删除属性：直接通过 obj.newProp = value 即可触发响应。
  3. 更细粒度的依赖追踪：基于属性的依赖收集，而非整个对象。

响应式流程：

  1. 依赖收集：当访问响应式数据时，触发 getter，将当前副作用（effect）收集为依赖。
  2. 触发更新：当修改数据时，触发 setter，通知所有关联的副作用重新执行。

单项数据流

单项数据流指的是数据的流动是单向的，数据的状态只能通过一个明确的方向进行传递和修改。在 Vue 应用中，数据通常是从父组件流向子组件，子组件不能直接修改从父组件接收到的数据，而是应该通过触发自定义事件通知父组件进行修改。

Vue 的单项数据流基于组件化的架构实现。每个组件都有自己独立的状态（数据），父组件可以通过 props 向子组件传递数据。当父组件的数据发生变化时，传递给子组件的 props 也会相应更新，从而触发子组件的重新渲染。但子组件对 props 的修改不会直接影响父组件的数据，这样就保证了数据流动的单向性。

  1. 避免数据混乱
    如果子组件可以直接修改父组件数据，多个子组件可能同时修改同一数据源，导致状态难以追踪。
  2. 明确数据所有权
    数据由父组件管理，子组件仅负责展示或触发操作，职责清晰。
  3. 提高可维护性
    数据流向单一，调试时更容易定位问题。

Vue 的单向数据流通过 props 和 emit 的强制约束，确保了组件之间的数据流动清晰可控。遵循这一原则可以大幅提升代码的可维护性，减少不可预知的副作用。在复杂场景中，合理使用状态管理工具（如 Pinia）是更好的选择。

watch

watch 函数用于监听一个或多个响应式数据源的变化，并在数据变化时执行回调函数。

  • immediate：设置为 true 时，回调函数会在 watch 被创建时立即执行一次。
  • deep：设置为 true 时，会深度监听对象内部属性的变化。

watchEffect 会立即执行传入的回调函数，并响应式地追踪回调函数中使用的所有响应式数据源，当这些数据源中的任何一个发生变化时，回调函数会再次执行。

watchEffect 会返回一个停止监听的函数，调用该函数可以停止对响应式数据源的监听。

**watch**** 和 **watchEffect** 的区别**

  • 触发时机：watch 默认不会立即执行回调函数，除非设置了 immediate: true；而 watchEffect 会立即执行回调函数。
  • 依赖追踪方式：watch 需要明确指定要监听的数据源；而 watchEffect 会自动追踪回调函数中使用的所有响应式数据源。
  • 回调参数：watch 的回调函数会接收新值和旧值作为参数；而 watchEffect 的回调函数不接收任何参数。

路由模式

Hash 模式（**createWebHashHistory**）

Hash 模式使用 URL 的哈希值（# 后面的部分）来模拟一个完整的 URL，当哈希值发生变化时，浏览器不会向服务器发送请求，而是触发 hashchange 事件，Vue Router 监听这个事件并根据哈希值的变化来渲染相应的组件。

  • 优点：兼容性好，支持所有浏览器，不需要服务器端进行任何配置。
  • 缺点：URL 中带有 # 符号，不够美观，而且在一些需要使用锚点功能的场景下可能会产生冲突。

History 模式（**createWebHistory**）

History 模式使用 HTML5 的 History API 来实现路由功能，通过 pushState 和 replaceState 方法改变浏览器的历史记录，从而实现无刷新的页面切换。当用户访问不同的 URL 时，浏览器会向服务器发送请求，因此需要服务器端进行相应的配置，以确保所有请求都能正确返回单页面应用的 HTML 文件。

  • 优点：URL 更加美观，没有 # 符号，符合传统的 URL 格式。
  • 缺点：需要服务器端进行配置，否则在刷新页面时可能会出现 404 错误。

3. Abstract 模式（**createMemoryHistory**）

Abstract 模式不依赖于浏览器的历史记录，它适用于非浏览器环境，如 Node.js 服务器端渲染（SSR）或小程序开发。在这种模式下，路由的切换通过内存中的历史记录来管理。

  • 优点：适用于非浏览器环境，提供了一种通用的路由管理方式。
  • 缺点：不能在浏览器中直接使用，主要用于特定的开发场景。

路由导航守卫是 Vue Router 提供的一种机制，用于在路由切换前后进行一些控制和处理，比如身份验证、权限检查、页面加载前的数据预获取等。Vue Router 提供了全局守卫、路由独享守卫和组件内守卫三种类型的导航守卫。

全局守卫

全局前置守卫

在路由跳转之前执行，可用于身份验证、权限检查等。

  • to：即将要进入的目标路由对象。
  • from：当前导航正要离开的路由对象。
  • next：一个函数，用于控制路由的跳转行为，必须调用该方法来 resolve 这个钩子。(已经不推荐)
  • 使用返回值
    • false : 取消当前导航
    • 默认或者undefined : 默认导航
    • 返回路由地址
      • string类型的path
      • 对象( path , query , params)

全局解析守卫

和 beforeEach 类似，但在导航被确认之前，同时所有组件内守卫和异步路由组件被解析之后调用。

全局后置钩子

在路由跳转之后执行，不接收 next 函数，不能改变导航本身。

路由独享守卫

路由独享守卫只作用于某个特定的路由配置，在路由配置中直接定义 beforeEnter 钩子。

**组件内守卫**

组件内守卫定义在路由组件内部，用于在组件层面进行导航控制。

**beforeRouteEnter**

在渲染该组件的对应路由被确认前调用，不能访问 this，因为组件实例还未创建。

**beforeRouteUpdate**

在当前路由改变，但是该组件被复用时调用，比如 /product/1 跳转到 /product/2。

**beforeRouteLeave**

在导航离开该组件的对应路由时调用，可用于阻止用户离开，比如表单未保存时提示用户。

导航解析流程
  1. 导航被触发。
  2. 在失活的组件里调用 beforeRouteLeave 守卫。
  3. 调用全局的 beforeEach 守卫
  4. 在重用的组件里调用 beforeRouteUpdate 守卫(2.2+)。
  5. 在路由配置里调用 beforeEnter。
  6. 解析异步路由组件。
  7. 在被激活的组件里调用 beforeRouteEnter。
  8. 调用全局的 beforeResolve 守卫(2.5+)。
  9. 导航被确认。
  10. 调用全局的 afterEach 钩子,
  11. 触发 DOM 更新。
  12. 调用 beforeRouteEnter 守卫中传给 next 的回调函数，创建好的组件实例会作为回调函数的参数传入

状态管理

VueX

在 Vue 中，Vuex 是一个专为 Vue.js 应用程序开发的状态管理模式。它采用集中式存储应用的所有组件的状态，并以相应的规则保证状态以一种可预测的方式发生变化。

安装和基本使用

安装

基本使用

在项目中创建一个 store 目录，并在其中创建 index.js 文件，用于配置和导出 Vuex store。

在 main.js 中引入并使用创建好的 store。

在模板中直接使用$store.state获取数据

在组合式api中使用useStore获取

Vuex 使用单一状态树:

  • 用一个对象就包含了全部的应用层级的状态 , 采用的是SSOT，Single Source of Truth，也可以翻译成单一数据源
  • 这也意味着，每个应用将仅仅包含一个 store 实例: 单状态树和模块化并不冲突,store中存在module

单一状态树的优势:

  • 如果你的状态信息是保存到多个Store对象中的，那么之后的管理和维护等等都会变得特别困难;所以Vuex也使用了单一状态树来管理应用层级的全部状态
  • 单一状态树能够让我们最直接的方式找到某个状态的片段:而且在之后的维护和调试过程中，也可以非常方便的管理和维护

Vuex-Store状态映射

Getters
  • 概念：getters 是 store 中的计算属性，用于获取 state 中的数据并进行处理，返回一个新的值。
  • 作用：可以对 state 中的数据进行封装和计算，避免在组件中重复编写相同的逻辑，提高代码的可维护性和复用性。

在创建 store 时，可以在 getters 对象中定义获取器函数。每个获取器函数接收 state 作为第一个参数，还可以接收其他 getters 作为第二个参数。

组合式 API 中，可以使用 useStore 函数来访问 store，进而访问 getters。

有时候，我们需要根据不同的条件来获取数据，这时可以让 getters 返回一个函数，从而实现接收参数的功能。

getter的映射

在选项式 API 中，可以通过 this.$store.getters 访问 getters。

也可以使用 mapGetters 辅助函数来简化代码

Mutation

mutation 是用于修改 store 中 state 的唯一途径，并且它必须是同步函数，这保证了状态变化的可追踪性和可预测性。

  • 概念：mutation 本质上是一个包含修改 state 逻辑的函数，它接收 state 作为第一个参数，还可以接收一个可选的 payload（即传递给 mutation 的额外参数）
  • 作用：集中管理状态的修改操作，使得状态的变化可追踪，方便调试和维护代码

在创建 store 时，在 mutations 对象中定义具体的 mutation 函数。

组合式 API 中，借助 useStore 函数访问 store，再使用 commit 方法触发 mutation。

为了增强代码的可读性和可维护性，通常会采用常量来定义 mutation 的名称

在组件中使用常量触发 mutation

Mutation的映射

在选项式 API 里，可以通过 this.$store.commit 方法来触发 mutation。

使用 mapMutations 辅助函数来简化代码

Actions

actions 是用来处理异步操作并最终提交 mutations 以修改 state 的。由于 mutations 必须是同步的，而异步操作（如网络请求）无法保证其同步性，所以就需要 actions 来处理这类情况

  • 概念：actions 是一些包含异步操作逻辑的函数，它不直接修改 state，而是通过提交 mutations 来间接修改 state。actions 接收一个与 store 实例具有相同方法和属性的 context 对象，通过它可以访问 state、getters 以及提交 mutations。
  • 作用：处理异步操作，如发起网络请求、定时器操作等，确保 state 的修改是可追踪和可预测的。

在创建 store 时，在 actions 对象中定义具体的 action 函数

组合式 API 中，利用 useStore 函数访问 store，再使用 dispatch 方法触发 action。

action 函数可以接收第二个参数作为 payload，用于传递额外的数据

组件中触发接收参数的 action

actions的映射

使用 mapActions 辅助函数来简化

Modules

Vuex 的 module（模块）允许将 store 分割成多个小模块，每个模块拥有自己独立的 state、mutations、actions 和 getters，使代码结构更加清晰、易于管理

  • 模块：模块是 Vuex 中用于组织代码的一种方式，将一个大的 store 拆分成多个小的模块，每个模块有自己独立的状态管理逻辑。
  • 命名空间：通过设置 namespaced: true，可以为模块开启命名空间，避免不同模块之间的命名冲突。

在store中引入

在组件中使用模块时，需要根据模块是否开启命名空间来决定访问方式

有时候，一个模块的 action 可能需要调用另一个模块的 action 或 mutation。可以通过 context.rootState、context.rootGetters、context.rootCommit 和 context.rootDispatch 来访问根 store 的状态和方法。

Pinia

Pinia 是一个轻量级的状态管理库，它提供了直观且可扩展的方式来管理应用程序的状态。

Getters

getters 类似于 Vue 组件中的计算属性，用于从 state 中派生数据。它们可以提供一种简洁且高效的方式来获取基于 state 的计算值，并且会自动响应 state 的变化。

在使用 defineStore 创建 store 时，可以通过 getters 选项来定义获取器。getters 是一个对象，其中每个属性都是一个函数，该函数接收 state 作为第一个参数。

Actions

actions 用于定义修改 state 的方法，它类似于 Vue 组件中的方法。actions 可以包含同步或异步操作，比如数据的获取、状态的更新等。

在使用 defineStore 创建 store 时，可以通过 actions 选项来定义行为。actions 是一个对象，其中每个属性都是一个函数，在函数内部可以使用 this 来访问 state、getters 以及其他 actions。

组件中，可以直接调用 store 中的 actions 方法。

actions 可以包含异步操作，比如发起网络请求。在 actions 中使用 async/await 语法可以方便地处理异步逻辑

在一个 action 中可以调用同一个 store 中的其他 actions。

actions 可以返回值，这在需要处理异步操作的结果或者链式调用 actions 时非常有用。
