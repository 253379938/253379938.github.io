动态加载图片

在 Vue 项目里，若 img 标签的 src 属性是动态的，可通过绑定 :src 来动态加载图片。下面会分别介绍基于 Webpack 和 Vite 的项目中动态加载图片的方式。

在 Webpack 项目中，需要借助 require 函数来动态加载图片

在 Webpack 里，require 是用于引入模块的函数。当使用 require 动态加载图片时，Webpack 会在打包阶段就对 require 语句进行解析，把图片文件处理成可以在浏览器中使用的 URL。因为 Webpack 在打包时会对整个项目进行静态分析，所以 require 能够准确地找到图片文件并处理它。

在 Vite 项目中，要使用 import.meta.glob 或者 new URL 来动态加载图片

Vite 是基于 ES 模块的开发服务器，它利用了现代浏览器原生支持 ES 模块的特性。在开发阶段，Vite 不会对项目进行打包，而是直接启动一个开发服务器，根据浏览器的请求按需加载模块。只有在生产环境下，Vite 才会进行打包。

new URL 是 JavaScript 原生的构造函数，用于创建一个 URL 对象。在 Vite 项目中，使用 new URL 可以根据当前模块的 URL 来生成图片的绝对 URL。因为 Vite 在开发阶段不会对项目进行打包，所以可以直接使用原生的 JavaScript 方法来处理图片路径。

import.meta.glob 是 Vite 提供的一个特殊功能，用于动态导入多个模块。在 Vite 项目中，使用 import.meta.glob 可以动态加载多个图片文件。import.meta.glob 会返回一个对象，对象的键是图片文件的路径，值是图片文件的模块。这样就可以根据图片的名称动态获取图片的 URL。
